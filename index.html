Endless Runner

by: Max Backer

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I chose to make this game because it helped me learn how to implement different types of movement functionalities. In addition, Endless Runners are very common so I wanted to see how challenging it was to program one. Programming this game also taught me how to use velocity control for character movement.

2. What I Learned

No user testing.

3. Architecture of My Game

My game involves obstacles that move in the player's direction. The player's objective is to jump when the obstacle approaches, and if they collide with it then the game goes to the end screen. I also added a background to the game screen to make it more interesting for the player. The game starts off with a Menu, and if you press Enter then you are transferred to the Game. Once you die you go to the End screen where you get the option play the game again by pressing Enter.

4. Technical Implementation

I am proud of the velocity control for the character because it makes the game more realistic. I struggled with player movement but I received some help on how to implement more realistic game movement for the player. This is an important aspect of the game because otherwise you would have unrealistic jumping mechanics.

5. Most Challenging Part

The most challenging part of this project was integrating velocity control because it required adding fake gravity to the game that made the character fall whenever they jumped. I was able to overcome this with help from my coach who guided me along the way.

6. Future Fixes and Updates

I would change the background because it doesn't perfectly match with the setting of the game. I would also improve the game by adjusting the obstacle collisions because they aren't perfectly aligned with the texture. If I had more time I might choose to add more options on the Menu screen, such as a settings button.
